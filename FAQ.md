## General Questions

### Q: Что такое Weather Runner?

**A:** Weather Runner - это бесконечный раннер, где игровая среда адаптируется к реальной погоде в вашем городе. Если на улице дождь - в игре будут лужи и скольжение, если снег - замедленное движение и снегопад.

### Q: Игра бесплатная?

**A:** Да! Weather Runner полностью бесплатен и имеет открытый исходный код (MIT License).

### Q: Нужно ли регистрироваться?

**A:** Нет, регистрация не требуется. Просто откройте файл и играйте.

### Q: На каких устройствах работает игра?

**A:** Игра работает на любых устройствах с современным браузером:
- Desktop (Windows, Mac, Linux)
- Mobile (iOS, Android)
- Tablet (iPad, Android tablets)

### Q: Какой браузер лучше использовать?

**A:** Рекомендуемые браузеры:
1. Google Chrome (лучшая производительность)
2. Mozilla Firefox
3. Safari (для macOS/iOS)
4. Microsoft Edge

## Gameplay

### Q: Как управлять персонажем?

**A:** 
- **Desktop:** Нажмите Пробел для прыжка
- **Mobile:** Тапните по экрану для прыжка

### Q: Как набрать больше очков?

**A:** Советы для высокого счёта:
1. Собирайте бонусы (+50 очков каждый)
2. Избегайте препятствий (иначе Game Over)
3. В дождь осторожно с лужами (ускорение)
4. В снег планируйте прыжки заранее (медленное падение)
5. В грозу следите за верхом экрана (молнии в воздухе)

### Q: Что происходит при столкновении?

**A:** Игра заканчивается мгновенно. Ваш счёт показывается вместе с рекордом.

### Q: Сохраняется ли прогресс?

**A:** Да, рекорд автоматически сохраняется в браузере (localStorage). Он будет доступен при следующем запуске.

### Q: Можно ли сохранить игру и продолжить позже?

**A:** Нет, это бесконечный раннер без системы сохранений. Каждая попытка начинается заново.

### Q: Есть ли уровни сложности?

**A:** Сложность зависит от погоды:
- ☀️ Ясно: Легко
- 🌧 Дождь: Средне
- ❄️ Снег: Средне
- ⛈ Гроза: Сложно

### Q: Почему персонаж скользит в дождь?

**A:** Это реалистичная механика - лужи делают поверхность скользкой, увеличивая скорость движения на 80%.

### Q: Можно ли отключить погодные эффекты?

**A:** В текущей версии нет, но это планируется в будущих обновлениях.

## Weather API

### Q: Зачем игре доступ к геолокации?

**A:** Чтобы определить ваше местоположение и получить актуальные данные о погоде через OpenWeatherMap API.

### Q: Безопасно ли предоставлять геолокацию?

**A:** Да. Геолокация используется только один раз при старте для запроса погоды. Координаты не сохраняются и не передаются третьим лицам.

### Q: Что если я откажу в доступе к геолокации?

**A:** Игра автоматически переключится в fallback режим и выберет случайную погоду из предустановленного списка.

### Q: Нужен ли интернет для игры?

**A:** Только для получения данных о погоде. Если интернета нет или API недоступен, игра использует fallback режим.

### Q: Почему погода в игре не совпадает с реальной?

**A:** Возможные причины:
1. API ключ не настроен (используется fallback)
2. Данные устарели (обновите страницу)
3. Погрешность геолокации

### Q: Где получить API ключ?

**A:** 
1. Зарегистрируйтесь на [openweathermap.org](https://openweathermap.org/api)
2. Перейдите в раздел "API Keys"
3. Скопируйте ваш ключ
4. Вставьте в `index.html` в переменную `API_KEY`

### Q: API ключ бесплатный?

**A:** Да, OpenWeatherMap предоставляет бесплатный тариф с лимитом 60 запросов/минуту и 1,000,000 запросов/месяц.

### Q: Можно ли использовать свой API ключ?

**A:** Да! Откройте `index.html`, найдите строку:
```javascript
this.API_KEY = 'YOUR_API_KEY';
```
Замените `YOUR_API_KEY` на ваш ключ.

## Technical Issues

### Q: Игра не загружается

**A:** Попробуйте:
1. Обновить страницу (Ctrl+R / Cmd+R)
2. Очистить кэш браузера
3. Попробовать другой браузер
4. Проверить консоль на ошибки (F12)

### Q: Браузер не запрашивает геолокацию

**A:** Проверьте настройки:
- **Chrome:** Настройки → Конфиденциальность → Настройки сайтов → Геолокация → Разрешить
- **Firefox:** Настройки → Приватность → Разрешения → Геолокация
- Убедитесь, что используете HTTPS или localhost

### Q: Игра лагает / тормозит

**A:** Решения:
1. Закройте другие вкладки
2. Обновите браузер до последней версии
3. Отключите расширения браузера
4. Проверьте нагрузку на CPU/GPU

### Q: Не работает прыжок по пробелу

**A:** Убедитесь, что:
1. Игра запущена (не на стартовом экране)
2. Фокус на окне игры
3. Не нажат Caps Lock или другие модификаторы

### Q: Рекорд не сохраняется

**A:** Проверьте:
1. Включён ли localStorage в браузере
2. Не включён ли режим инкогнито (localStorage может быть отключён)
3. Достаточно ли места для хранения

### Q: Игра выглядит размытой

**A:** Это может быть связано с масштабированием экрана. Попробуйте:
1. Установить масштаб 100% в браузере (Ctrl+0)
2. Использовать полноэкранный режим (F11)

### Q: Не отображаются эмодзи погоды

**A:** Обновите операционную систему и браузер. Старые версии могут не поддерживать новые эмодзи.

### Q: Белый экран вместо игры

**A:** Откройте консоль (F12) и проверьте ошибки. Чаще всего проблема в:
1. JavaScript ошибках
2. Блокировке скриптов расширениями
3. Проблемах с Canvas API

### Q: Можно ли играть офлайн?

**A:** Да, но только в fallback режиме со случайной погодой. Для реальной погоды нужен интернет.

## Development

### Q: На чём написана игра?

**A:** Чистый JavaScript (ES6+), HTML5 Canvas, CSS3. Никаких фреймворков или библиотек.

### Q: Почему не использован Three.js?

**A:** Для 2D раннера Three.js был бы избыточен. Canvas 2D проще, легче и быстрее для этого типа игры.

### Q: Как изменить скорость игры?

**A:** В `index.html` найдите:
```javascript
this.baseSpeed = 5;
```
Измените значение (больше = быстрее).

### Q: Как добавить новую погоду?

**A:** См. раздел "Расширение игры" в [DEVELOPER_NOTES.md](./DEVELOPER_NOTES.md).

### Q: Можно ли изменить размер Canvas?

**A:** Да, измените атрибуты width и height у canvas и обновите соответствующие константы в коде.

### Q: Как добавить звуки?

**A:** Используйте Web Audio API:
```javascript
const jumpSound = new Audio('jump.mp3');
jumpSound.play();
```

### Q: Где хранится код игры?

**A:** Весь код в одном файле `index.html` в тегах `<script>`.

### Q: Можно ли разделить код на файлы?

**A:** Да, вынесите классы в отдельные `.js` файлы и подключите через `<script src="...">`.

### Q: Как протестировать разные погоды?

**A:** Временно измените `mapWeatherToGameState()` чтобы возвращать нужную погоду:
```javascript
mapWeatherToGameState(code) {
    return 'STORM'; // Всегда гроза для теста
}
```

### Q: Есть ли unit тесты?

**A:** В текущей версии нет, но структура кода позволяет легко их добавить.

### Q: Как оптимизировать производительность?

**A:** См. раздел "Performance Optimization" в [DEVELOPER_NOTES.md](./DEVELOPER_NOTES.md).

## Advanced Questions

### Q: Как работает AABB коллизия?

**A:** Axis-Aligned Bounding Box проверяет пересечение двух прямоугольников:
```javascript
rect1.x < rect2.x + rect2.width &&
rect1.x + rect1.width > rect2.x &&
rect1.y < rect2.y + rect2.height &&
rect1.y + rect1.height > rect2.y
```

### Q: Почему используется requestAnimationFrame?

**A:** 
- Синхронизация с частотой обновления монитора
- Автоматическая пауза в неактивных вкладках
- Плавная анимация без tearing
- Оптимальная производительность

### Q: Как рассчитать FPS?

**A:** 
```javascript
let lastTime = performance.now();
let frameCount = 0;

function gameLoop() {
    frameCount++;
    const currentTime = performance.now();
    if (currentTime - lastTime >= 1000) {
        console.log(`FPS: ${frameCount}`);
        frameCount = 0;
        lastTime = currentTime;
    }
    requestAnimationFrame(gameLoop);
}
```

### Q: Можно ли добавить мультиплеер?

**A:** Теоретически да, но потребуется:
- WebSocket сервер
- Синхронизация состояний
- Компенсация лагов
- Серверная валидация

### Q: Как добавить AI противников?

**A:** Создайте класс `Enemy` с простым AI:
```javascript
class Enemy {
    update() {
        // Простое поведение: двигаться влево
        this.x -= this.speed;
        
        // Или более сложное: избегать препятствий
        if (this.detectObstacle()) {
            this.jump();
        }
    }
}
```

### Q: Как реализовать procedurally generated levels?

**A:** Weather Runner уже использует procedural generation для спавна препятствий на основе погоды.
