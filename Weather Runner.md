# Weather Runner: Полный план разработки (HTML/CSS/JS)

## 1. Общая концепция и архитектура
Создать игру в жанре "бесконечный раннер" на HTML5 Canvas. Ключевая особенность: **синхронизация игровых механик и визуала с реальной погодой в городе пользователя** через API OpenWeatherMap.

**Структура файлов (в одном HTML):**
- `<style>`: Стили затемнения, расположение Canvas и UI.
- `<canvas>`: Игровое поле.
- `<div>`: Оверлей с данными о погоде и кнопкой "Играть".
- `<script>`: Вся логика игры.

## 2. Интеграция реальной погоды (Weather API)

### 2.1. Используемый сервис
**OpenWeatherMap Current Weather Data API** (бесплатный тариф).
*Ключ API: Запросить у пользователя или вставить тестовый. В коде оставить переменную `const API_KEY = 'ВАШ_КЛЮЧ';`*

### 2.2. Алгоритм определения погоды для игры
1. Получить координаты пользователя через `navigator.geolocation`.
2. Запросить `api.openweathermap.org/data/2.5/weather?lat={lat}&lon={lon}&appid={key}&units=metric`.
3. Парсинг ответа в объект `GameWeather`:
   - `main.temp` -> Температура (фон цвета).
   - `weather[0].id` -> Код погоды.

### 2.3. Сопоставление кодов погоды с игровыми состояниями (Логика)
Реализовать функцию `mapWeatherToGameState(weatherCode)`:
- **Clear** (800, 801): `SUNNY`
- **Rain / Drizzle** (200-531): `RAIN`
- **Snow** (600-622): `SNOW`
- **Thunderstorm** (200-232): `STORM`

### 2.4. Fallback (Резервный режим)
Если геолокация запрещена или API не отвечает:
- Случайный выбор погоды.
- Или установка `SUNNY` по умолчанию с сообщением в консоли.

## 3. Игровая механика на Canvas (Детали реализации)

### 3.1. Базовый цикл (Game Loop)
Использовать `requestAnimationFrame`.
Скорость мира: `baseSpeed = 5`.
Пол: фиксированная линия `groundY = 350`.

### 3.2. Персонаж (Игрок)
- Размер: 30x30 px.
- Позиция X: фиксированная (`playerX = 100`).
- Физика: Гравитация (`gravity = 0.5`), Прыжок (`jumpForce = -10`).
- Условие проигрыша: Столкновение с препятствием (коллизия AABB - прямоугольник с прямоугольником).

### 3.3. Система препятствий и эффектов погоды (Самое важное)

Класс `Obstacle`:
- Поля: `x`, `y`, `width`, `height`, `type` (лужа, молния, сугроб).

**Модификаторы в зависимости от состояния `currentWeather`:**

| Состояние | Визуал фона | Частота спавна | Тип препятствия | Спецэффект |
| :--- | :--- | :--- | :--- | :--- |
| **SUNNY** | Ярко-голубой | Низкая | Камни (обычные) | Нет |
| **RAIN** | Серый | Средняя | **Лужи** | **Скольжение**: если игрок приземляется на лужу, `speed` мира временно умножается на 1.5. |
| **SNOW** | Белый/Голубой | Низкая | **Сугробы** | **Замедление**: `baseSpeed = 3`, гравитация меньше (`gravity = 0.3`). |
| **STORM** | Темно-фиолетовый | Высокая | **Молнии** | **Падение с неба**: Препятствия появляются не только на земле, но и **в воздухе** (Y = 200-300). |

### 3.4. Бонусы
- Тип: Звезда / Зонт (в зависимости от погоды).
- Эффект: Увеличение счета на +50 (вместо +1 за кадр).
- Сбор: Уничтожение при касании игрока.

## 4. Пользовательский интерфейс (UI)

### 4.1. Стартовый экран
- Кнопка **"Узнать погоду и играть"**.
- Лоадер (спиннер) на время запроса к API.

### 4.2. Игровой HUD (Поверх Canvas)
- **Левый верхний угол:** Иконка текущей погоды (☀️, 🌧, ❄️, ⛈) + реальная температура (°C).
- **Правый верхний угол:** Счетчик дистанции (в метрах или очках).
- **Эффекты:** При наложении эффекта ("Скользко!", "Замедление!") показывать текстовое всплывающее уведомление над персонажем на 1 секунду.

### 4.3. Экран Game Over
- Показать итоговый счет.
- Показать рекорд (сохранять в `localStorage`).
- Кнопка **"Попробовать снова"** (повторный запрос погоды).

## 5. Визуализация и Рендеринг

**Стиль:** Flat Design с пиксельной графикой (удобно для Canvas).
- **Игрок:** Синий квадрат с глазом или простой SVG-стиль `fillRect`.
- **Лужа:** Полупрозрачный синий эллипс (`fillStyle = 'rgba(0, 0, 255, 0.4)'`).
- **Молния:** Желтая зигзагообразная линия или вертикальный прямоугольник.
- **Снег:** Частицы (в классе `SnowParticle`), фон меняется на градиент.

## 6. Технические требования к коду (для нейросети)

1.  **Весь код должен быть в одном файле `index.html`.**
2.  **Обработка ввода:** Только **Пробел** (прыжок). Запретить скролл страницы по пробелу (`preventDefault`).
3.  **Адаптивность:** Canvas размер `800x400`. Для мобильных: можно добавить кнопку "Прыжок" под канвасом, но в приоритете десктоп.
4.  **Классы:** Использовать синтаксис ES6 классов:
    - `class Player {}`
    - `class Obstacle {}`
    - `class Collectible {}`
    - `class Game {}`
    - `class WeatherService {}`
